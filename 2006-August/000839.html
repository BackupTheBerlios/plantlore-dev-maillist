Summary: Use hash function for stroing passwords

Details: In  order to improve security, passwords should not be saved in the database as plaintext. Message digest (such as MD5 or SHA) should be used instead. This way, after user submits a password, digest (hash) is created and sent to the server, where it is compared with digest saved in the database. See java.security.MessageDigest for implementation details.
Affected modules: Login, UserManager
