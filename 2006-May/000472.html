Comment:
It is because the "Add" model listens to the AutoTextArea's FocusLost events only. Thus, if the AutoTextArea did not get the focus, the model was never notified - which is why it assumed no Plants were entered.

The AutoTextArea sends PropertyChanged event every time a line is changed/added/deleted. Add/Edit dialog should reflect that.
