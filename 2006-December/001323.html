*Problém:  Jestliže spustím server bez gui, tak se k němu nepřipojím jiným
plantlore-serverem z jiné mašiny*

*Reprodukce:*
Nepodařilo se reprodukovat.
Server jsem spustil z příkazové řádky, připojení jsem provedl grafickým
rozhraním.
Nechal jsem si obnovit seznam klientů (žádní nebyli) a poté jsem server
vzdáleně vypnul.
Problémy se neobjevily.

*Navrhované řešení:*

   - Zkontrolovat nastavení serveru (port, heslo).
   - Zkontrolovat nastavení práv (*java.policy*).

V případě potíží dodat výpis logu z obou serverů.



*Problém: **Nedaří se mi spustit plantlore-server při startu linuxu bez
konzole z
rc.local příkazem **java -Djava.rmi.server.hostname=10.1.69.1 -jar
/usr/local/Plantlore/plantlore-server.jar port 2332 nogui**.
*
*Reprodukce:*
Nereprodukováno.
Neprovozuji linux ani nemám přístup k žádnému linuxovému stroji s
dostatečnými právy,
abych mohl problém reprodukovat.
Podle Google jde však spíš o chybu samotné Javy;
krom toho java.util.Queue, která je postrádána, je součást standardních
knihoven Javy.
Chyba není v Plantlore.

*Navrhované řešení:*
 Vyhledat si:

   - java.lang.Class.initializeClass(libgcj.so.7rh)
   - java.lang.ClassNotFoundException: java.util.Queue not found in
   gnu.gcj.runtime.SystemClassLoader
   - java.net.URLClassLoader.findClass(libgcj.so.7rh )

Zařídit se podle návodů k linuxu&javy.
